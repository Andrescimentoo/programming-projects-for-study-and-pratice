                                                            N8N
O N8N é uma das principais ferramentas de automação usada na web para automatizar processos e resolver problemas reais
com ela podemos fazer integrações com sistemas de terceiros, integrar chatbots  e construir fluxogramas automatizados para empresas

                                                        Condiiconais no N8N
Aprendi que num fluxograma, quando queremos que nosso fluxo tome rumos diferentes a depender de eventos distintos, usamos condicionais
para dividir o fluxo do workFlow. Entretanto o N8N nos oferece a princípio duas condicionais para dividirmos os caminhos que nosso fluxo executará
Nos oferece o nó condicional IF e o nó condicional Switch.
                                                          IF vs SWITCH          
Aprendi que usamos IF quando nosso sistema tem apenas 2 saidas finais no workflow. ex: um webhook (ouvido do fluxograma que escuta chamadas externas
recebendo dados, objetos e etc...) escuta 2 chamadas. Uma get e uma Post, se for get, um nó é executado. Se for Post outro nó é executado
Usamos If quando existem apenas 2 saídas possiveis para o fluxograma. E se forem mais de 2 saídass?? ou seja quando se trata de uma automação maior
com mais etapas? Aí entra o nó Switch! que é resposável e mais útil quando se trata de um workFlow com mais de 2 saídas possiveis
